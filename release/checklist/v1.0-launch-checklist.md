## Rollback Triggers

### Immediate Rollback (Stop Launch)

If any of these occur, immediately rollback:

- [ ] Error rate >5% for 5 minutes
- [ ] P95 latency >2000ms for 5 minutes
- [ ] Database connection failures
- [ ] Authentication system down
- [ ] Data corruption detected
- [ ] Security vulnerability exposed
- [ ] Critical user impact reported

### Rollback Procedure

```bash
# 1. Alert team
# Post in #incident-response channel
# Page on-call engineer

# 2. Execute rollback
npm run rollback:production

# 3. Verify rollback
npm run verify:production

# 4. Monitor stability
npm run metrics:collect

# 5. Communicate status
# Update status page
# Notify stakeholders
# Post incident summary
```

## Incident Response

### Severity Levels

#### SEV1 - Critical (System Down)
- **Response Time:** <15 minutes
- **Resolution Target:** <1 hour
- **Communication:** Public status page updated
- **Escalation:** CTO notified immediately

#### SEV2 - High (Major Degradation)
- **Response Time:** <30 minutes
- **Resolution Target:** <4 hours
- **Communication:** Email to affected users
- **Escalation:** VP Engineering notified

#### SEV3 - Medium (Minor Issues)
- **Response Time:** <1 hour
- **Resolution Target:** <24 hours
- **Communication:** Internal tracking
- **Escalation:** Engineering Manager notified

#### SEV4 - Low (Cosmetic/Documentation)
- **Response Time:** <4 hours
- **Resolution Target:** <1 week
- **Communication:** Backlog item
- **Escalation:** Team Lead notified
