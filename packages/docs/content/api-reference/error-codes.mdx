# Error Codes

Reference for all error codes returned by the ClaudeFlare API, including causes, solutions, and example responses.

## Error Response Format

All errors follow this format:

```typescript
interface ErrorResponse {
  error: {
    code: string;           // Machine-readable error code
    message: string;        // Human-readable error message
    details?: any;          // Additional error details
    requestId?: string;     // Request identifier for support
    timestamp: number;      // Unix timestamp
  };
}
```

## Authentication Errors

### `INVALID_API_KEY`

The provided API key is invalid or expired.

**Status:** 401

**Example:**
```json
{
  "error": {
    "code": "INVALID_API_KEY",
    "message": "The provided API key is invalid or has been revoked",
    "timestamp": 1234567890
  }
}
```

**Solutions:**
- Verify API key is correct
- Check key hasn't been revoked
- Regenerate API key if needed

### `MISSING_API_KEY`

No API key provided in request.

**Status:** 401

**Example:**
```json
{
  "error": {
    "code": "MISSING_API_KEY",
    "message": "API key is required. Include it in the Authorization header.",
    "timestamp": 1234567890
  }
}
```

**Solutions:**
- Add Authorization header: `Authorization: Bearer YOUR_API_KEY`

### `INSUFFICIENT_PERMISSIONS`

API key doesn't have required permissions.

**Status:** 403

**Example:**
```json
{
  "error": {
    "code": "INSUFFICIENT_PERMISSIONS",
    "message": "Your API key doesn't have permission to access this resource",
    "required": ["agents:write"],
    "current": ["agents:read"],
    "timestamp": 1234567890
  }
}
```

**Solutions:**
- Check API key permissions
- Use API key with appropriate scope
- Contact support for permission upgrade

## Rate Limit Errors

### `RATE_LIMITED`

Too many requests sent.

**Status:** 429

**Example:**
```json
{
  "error": {
    "code": "RATE_LIMITED",
    "message": "Rate limit exceeded. Please retry after 60 seconds.",
    "limit": 100,
    "remaining": 0,
    "reset": 1234567950,
    "retryAfter": 60,
    "timestamp": 1234567890
  }
}
```

**Solutions:**
- Implement exponential backoff
- Check `retryAfter` header
- Reduce request frequency
- Consider upgrading plan

### `QUOTA_EXCEEDED`

Monthly quota exceeded.

**Status:** 429

**Example:**
```json
{
  "error": {
    "code": "QUOTA_EXCEEDED",
    "message": "Monthly quota exceeded. Please upgrade your plan.",
    "quota": {
      "limit": 10000,
      "used": 10000,
      "remaining": 0,
      "resets": "2026-02-01T00:00:00Z"
    },
    "timestamp": 1234567890
  }
}
```

**Solutions:**
- Wait for quota reset
- Upgrade plan
- Optimize usage

## Error Handling Best Practices

### 1. Always Check Error Codes

```typescript
try {
  const response = await client.chat.completions.create(request);
} catch (error) {
  if (error.code === 'RATE_LIMITED') {
    // Implement backoff
    await sleep(error.retryAfter * 1000);
  } else if (error.code === 'QUOTA_EXCEEDED') {
    // Handle quota exceeded
    alert('Monthly quota exceeded');
  } else {
    // Handle other errors
    console.error('Error:', error.message);
  }
}
```

### 2. Use Request IDs for Support

```typescript
catch (error) {
  if (error.requestId) {
    console.error('Error ID:', error.requestId);
    // Include in support request
  }
}
```

### 3. Implement Retry Logic

```typescript
async function retryWithBackoff(fn, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fn();
    } catch (error) {
      if (error.code === 'RATE_LIMITED' && i < maxRetries - 1) {
        const delay = Math.pow(2, i) * 1000;
        await sleep(delay);
        continue;
      }
      throw error;
    }
  }
}
```

### 4. Log Errors for Debugging

```typescript
catch (error) {
  console.error({
    code: error.code,
    message: error.message,
    details: error.details,
    timestamp: error.timestamp,
    requestId: error.requestId
  });
}
```
