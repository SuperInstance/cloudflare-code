# Agents API

The Agents API enables sophisticated multi-agent workflows where specialized AI agents collaborate to complete complex coding tasks. Built on Cloudflare Durable Objects for stateful, real-time agent coordination.

## Overview

ClaudeFlare's agent system consists of specialized agents:

- **Director Agent**: Orchestrates workflows and delegates tasks
- **Planner Agent**: Breaks down complex tasks into steps
- **Executor Agent**: Executes code generation and modifications
- **Reviewer Agent**: Reviews code for quality and security
- **Analyzer Agent**: Analyzes codebases and provides insights
- **Messenger Agent**: Handles multi-party communication

## Endpoints

### Orchestrate Agents

Execute multi-agent workflows for complex tasks.

**Endpoint:** `POST /v1/agents/orchestrate`

**Authentication:** Bearer token required

#### Request

```typescript
interface AgentOrchestrationRequest {
  // Required
  sessionId: string;         // Unique session identifier
  userId: string;            // User identifier
  messages: Array<{
    role: 'user' | 'assistant' | 'system';
    content: string;
    timestamp?: number;
  }>;

  // Optional
  context?: {
    codebaseId?: string;     // Codebase context
    files?: string[];        // Specific files to analyze
    language?: string;       // Programming language
    framework?: string;      // Framework being used
  };

  preferences?: {
    agentStyle?: 'concise' | 'detailed' | 'tutorial';
    includeTests?: boolean;
    includeDocs?: boolean;
    verifySecurity?: boolean;
  };

  // Workflow configuration
  workflow?: 'auto' | 'code-review' | 'refactor' | 'debug' | 'feature-add';
  maxSteps?: number;         // Maximum workflow steps (default: 10)
  timeout?: number;          // Timeout in seconds (default: 300)
}
```

#### Example Request

```bash

      "agentStyle": "detailed",

  }'
```

#### Response

```typescript
interface AgentOrchestrationResponse {
  sessionId: string;
  status: 'pending' | 'running' | 'completed' | 'failed';

  // Agent workflow
  workflow: {
    steps: Array<{
      stepId: string;
      agent: string;         // Agent type (planner, executor, reviewer, etc.)
      action: string;        // Action performed
      status: 'pending' | 'running' | 'completed' | 'failed';
      input: any;            // Input to this step
      output?: any;          // Output from this step
      timestamp: number;
      duration?: number;     // Duration in ms
    }>;
    currentStep: number;
    totalSteps: number;
  };

  // Results (available when status is 'completed')
  results?: {
    message: {
      role: 'assistant';
      content: string;
    };
    artifacts?: Array<{
      type: 'file' | 'code' | 'test' | 'documentation';
      path?: string;
      content: string;
      language?: string;
    }>;
    changes?: Array<{
      type: 'create' | 'update' | 'delete';
      path: string;
      diff?: string;
    }>;
  };

  // Metrics
  metrics: {
    latency: number;         // Total orchestration time
    agentCalls: number;      // Number of agent invocations
    tokensUsed: number;
    providers: string[];     // Providers used
  };

  // Errors
  error?: {
    code: string;
    message: string;
    step?: string;
  };

  timestamp: number;
}
```

### List Available Agents

Get list of available agents by type.

**Endpoint:** `GET /v1/agents/available/:type?`

#### Parameters

- `type` (optional): Agent type filter (`director`, `planner`, `executor`, `reviewer`, `analyzer`, `messenger`)

#### Response

```typescript
interface AvailableAgentsResponse {

    available: boolean;
    load: number;

  count: number;
  timestamp: number;
}
```

## Agent Communication

Agents communicate through a messaging system with:

- **Direct Messages**: Point-to-point communication
- **Broadcast**: One-to-many communication
- **Request/Response**: Synchronous communication patterns
- **Pub/Sub**: Asynchronous event handling

## State Management

Agent state is managed through Cloudflare Durable Objects:

- **Session State**: Maintained across agent interactions
- **Context Window**: Shared context for all agents in workflow
- **Checkpointing**: Progress saved at each step
- **Recovery**: Automatic recovery from failures

## Error Handling

### Error Codes

| Code | Description |
|------|-------------|
| `WORKFLOW_TIMEOUT` | Workflow exceeded timeout |
| `AGENT_UNAVAILABLE` | Required agent not available |
| `STEP_FAILED` | Workflow step failed |
| `CONTEXT_TOO_LARGE` | Context exceeds agent capacity |
| `RATE_LIMITED` | Too many agent requests |

### Error Response

```typescript
interface AgentErrorResponse {
  error: {
    code: string;
    message: string;
    step?: string;          // Failed step
    agent?: string;         // Failed agent
    recoverable: boolean;   // Can workflow continue?
    retryAfter?: number;    // Seconds before retry
  };
  sessionId: string;
  timestamp: number;
}
```

## Best Practices

1. **Set Appropriate Timeouts**: Complex workflows may take time
2. **Provide Context**: Include relevant files and codebase info
3. **Use Specific Workflows**: Better than auto for known tasks
4. **Monitor Progress**: Check status for long-running workflows
5. **Handle Errors Gracefully**: Implement retry logic for recoverable errors

## SDK Examples

### TypeScript

```typescript
import { ClaudeFlare } from '@claudeflare/sdk';

const client = new ClaudeFlare({ apiKey: 'your-api-key' });

// Orchestrate agents
const result = await client.agents.orchestrate({
  sessionId: 'sess_123',
  userId: 'user_456',
  messages: [
    {
      role: 'user',
      content: 'Add user authentication'
    }
  ],
  workflow: 'feature-add'
});

// Monitor progress
while (result.status === 'running') {
  await new Promise(resolve => setTimeout(resolve, 1000));
  const status = await client.agents.getStatus(result.sessionId);
  console.log('Progress:', status.workflow.currentStep, '/', status.workflow.totalSteps);
}

// Get results
console.log(result.results);
```

### Python

```python
import asyncio
from claudeflare import ClaudeFlare

client = ClaudeFlare(api_key='your-api-key')

async def orchestrate_workflow():
    result = await client.agents.orchestrate(
        session_id='sess_123',
        user_id='user_456',
        messages=[{
            'role': 'user',
            'content': 'Add user authentication'
        }],
        workflow='feature-add'
    )

    # Monitor progress
    while result.status == 'running':
        await asyncio.sleep(1)
        status = await client.agents.get_status(result.session_id)
        print(f"Progress: {status.workflow.current_step}/{status.workflow.total_steps}")

    return result

result = asyncio.run(orchestrate_workflow())
```
