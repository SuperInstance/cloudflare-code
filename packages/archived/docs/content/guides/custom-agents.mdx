# Custom Agents

Custom agents allow you to create specialized AI agents tailored to your specific use cases. Define custom behaviors, tools, and workflows to extend ClaudeFlare's capabilities.

## Overview

ClaudeFlare supports several types of custom agents:

- **Tool Agents**: Agents with specific tools and capabilities
- **Workflow Agents**: Agents that follow predefined workflows
- **Domain Agents**: Agents specialized in specific domains
- **Composite Agents**: Agents that combine multiple sub-agents

## Creating Custom Agents

### Agent Definition

Define your custom agent with a configuration object:

```typescript
interface CustomAgentConfig {
  // Identity
  name: string;                  // Unique agent name
  type: 'tool' | 'workflow' | 'domain' | 'composite';
  version: string;               // Version semver

  // Description
  description: string;
  capabilities: string[];        // List of capabilities

  // Behavior
  systemPrompt: string;          // System prompt for the agent
  temperature?: number;          // Default temperature (0-2)
  maxTokens?: number;            // Default max tokens

  // Tools (for tool agents)
  tools?: Array<{
    name: string;
    description: string;
    parameters: any;
    handler: string;             // Reference to handler function
  }>;

  // Workflow (for workflow agents)
  workflow?: {
    steps: Array<{
      name: string;
      agent?: string;            // Sub-agent to use
      action: string;
      input: any;
      output?: any;
    }>;
  };

  // Sub-agents (for composite agents)
  subAgents?: string[];          // Agent IDs to delegate to

  // Configuration
  config?: {
    provider?: 'auto' | 'cloudflare' | 'groq' | 'cerebras';
    model?: string;
    timeout?: number;
    retryAttempts?: number;
  };
}
```

### Example: Code Review Agent

Create a specialized code review agent:

```typescript
const codeReviewAgent: CustomAgentConfig = {
  name: 'code-reviewer',
  type: 'domain',
  version: '1.0.0',

  capabilities: [
    'security-analysis',
    'performance-analysis',
    'best-practices-check',
    'code-style-review'
  ],
  systemPrompt: `You are a specialized code review agent with expertise in:
- Security vulnerabilities (OWASP Top 10, common exploits)

- Language-specific best practices

Review code thoroughly and provide:
1. Security issues with severity ratings
2. Performance concerns with optimization suggestions
3. Best practice violations with improvements
4. Code style recommendations

Be constructive, specific, and provide examples.`,
  temperature: 0.3,
  maxTokens: 2000,
  config: {
    provider: 'auto',
    model: '@cf/meta/llama-3.1-8b-instruct',
    timeout: 60
  }
};
```

### Example: Workflow Agent

Create an agent that follows a specific workflow:

```typescript
const refactoringAgent: CustomAgentConfig = {
  name: 'refactor-orchestrator',
  type: 'workflow',
  version: '1.0.0',
  description: 'Orchestrates code refactoring workflow',
  capabilities: [
    'code-analysis',
    'refactoring-planning',
    'code-transformation',
    'verification'
  ],
  systemPrompt: `You are a refactoring orchestrator. Guide code through:
1. Analysis phase: Identify issues and improvements
2. Planning phase: Create detailed refactoring plan
3. Transformation phase: Apply refactoring changes
4. Verification phase: Ensure tests pass and behavior preserved`,
  workflow: {
    steps: [
      {
        name: 'analyze',
        agent: 'code-analyzer',
        action: 'analyze-code',
        input: {
          code: '{code}',
          goals: '{goals}'
        }
      },
      {
        name: 'plan',
        agent: 'planner',
        action: 'create-plan',
        input: {
          analysis: '{analyze.output}',
          goals: '{goals}'
        }
      },
      {
        name: 'transform',
        agent: 'executor',
        action: 'apply-refactoring',
        input: {
          plan: '{plan.output}',
          code: '{code}'
        }
      },
      {
        name: 'verify',
        agent: 'reviewer',
        action: 'verify-refactoring',
        input: {
          original: '{code}',
          refactored: '{transform.output}'
        }
      }
    ]
  }
};
```

## Registering Custom Agents

### Register via API

```bash
curl -X POST https://api.claudeflare.com/v1/agents/custom \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d @code-review-agent.json
```

### Register via SDK

```typescript
import { ClaudeFlare } from '@claudeflare/sdk';

const client = new ClaudeFlare({ apiKey: 'your-api-key' });

const agent = await client.agents.register({
  name: 'code-reviewer',
  type: 'domain',
  version: '1.0.0',
  description: 'Specialized code review agent',
  capabilities: ['security-analysis', 'performance-analysis'],
  systemPrompt: 'You are a code review expert...',
  temperature: 0.3
});

console.log('Agent registered:', agent.id);
```

## Agent Tools

### Built-in Tools

ClaudeFlare provides built-in tools:

- **file-read**: Read files from codebase
- **file-write**: Write files to codebase
- **code-search**: Search codebase
- **execute-command**: Execute shell commands (sandboxed)
- **run-tests**: Execute test suite
- **analyze-dependencies**: Analyze project dependencies

### Custom Tools

Define custom tools:

```typescript
const customTool = {
  name: 'check-security',
  description: 'Check code for security vulnerabilities',
  parameters: {
    type: 'object',
    properties: {
      code: { type: 'string', description: 'Code to check' },
      language: { type: 'string', description: 'Programming language' }
    },
    required: ['code', 'language']
  },
  handler: async (params) => {
    // Implement security check logic
    const issues = [];

    // SQL injection check
    if (params.code.includes('SELECT * FROM')) {
      issues.push({
        type: 'sql-injection',
        severity: 'high',
        line: findLineNumber(params.code, 'SELECT')
      });
    }

    // XSS check
    if (params.code.includes('innerHTML')) {
      issues.push({
        type: 'xss',
        severity: 'medium',
        line: findLineNumber(params.code, 'innerHTML')
      });
    }

    return {
      issues,
      summary: `Found ${issues.length} security issues`
    };
  }
};
```

## Best Practices

### 1. Clear Capabilities

Define specific, actionable capabilities:

```typescript
capabilities: [
  'sql-injection-detection',  // Good
  'code-analysis'             // Too vague
]
```

### 2. Structured System Prompts

Use structured prompts with clear instructions:

```typescript
systemPrompt: `
Role: Security Analyst
Expertise: OWASP Top 10, Common Vulnerabilities

Process:
1. Scan for known vulnerability patterns
2. Analyze data flow and input handling
3. Check authentication and authorization
4. Review cryptography usage

Output Format:
- Severity rating (Critical/High/Medium/Low)
- Vulnerability type
- Location (file:line)
- Remediation steps
- Code example of fix
`
```

### 3. Appropriate Temperature

Use lower temperatures for code analysis (0.2-0.4), higher for creative tasks (0.7-1.0).

### 4. Tool Composition

Combine tools for complex workflows:

```typescript
tools: [
  analyzeStructure,
  checkSecurity,
  verifyPerformance,
  generateTests
]
```

### 5. Error Handling

Implement robust error handling in tool handlers:

```typescript
handler: async (params) => {
  try {
    return await executeTool(params);
  } catch (error) {
    return {
      success: false,
      error: error.message,
      recovery: 'Try with different parameters'
    };
  }
}
```

## Troubleshooting

### Agent Not Responding

- Check agent is registered
- Verify agent is healthy
- Review agent logs
- Check timeout settings

### Tool Execution Failing

- Verify tool parameters
- Check tool handler implementation
- Review error messages
- Test tool in isolation

### Workflow Stuck

- Check workflow step definitions
- Verify agent dependencies
- Review step outputs
- Check for circular dependencies
