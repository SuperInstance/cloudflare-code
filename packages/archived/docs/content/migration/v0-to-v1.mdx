# Migrating from v0 to v1

This guide helps you upgrade from ClaudeFlare v0.x to v1.0.

## Overview

ClaudeFlare v1.0 is a major release with breaking changes and new features:

- **Multi-provider routing** with automatic failover
- **Durable Objects** for stateful workflows
- **Enhanced SDKs** with better error handling
- **New API endpoints** for advanced features
- **Improved performance** with edge caching

## Breaking Changes

### 1. API Endpoint Changes

#### v0.x

```typescript
const response = await fetch('https://api.claudeflare.com/chat', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${apiKey}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    messages: [{ role: 'user', content: 'Hello' }]
  })
});
```

#### v1.0

```typescript
const response = await fetch('https://api.claudeflare.com/v1/chat', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${apiKey}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    messages: [{ role: 'user', content: 'Hello' }]
  })
});
```

**Change:** All endpoints now use `/v1` prefix for versioning.

**Migration:** Update base URL or use SDK which handles this automatically.

### 2. Authentication

#### v0.x

```typescript
const client = new ClaudeFlare({
  apiKey: 'your-key'
});
```

#### v1.0

```typescript
const client = new ClaudeFlare({
  apiKey: 'your-key',
  endpoint: 'https://api.claudeflare.com/v1'
});
```

**Change:** Explicit endpoint configuration required.

### 3. Response Format

#### v0.x

```json
{
  "message": {
    "role": "assistant",
    "content": "Hello!"
  }
}
```

#### v1.0

```json
{
  "id": "chatcmpl-123",
  "object": "chat.completion",
  "created": 1234567890,
  "model": "@cf/meta/llama-3.1-8b-instruct",
  "choices": [{
    "index": 0,
    "message": {
      "role": "assistant",
      "content": "Hello!"
    },
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 10,
    "completion_tokens": 5,
    "total_tokens": 15
  },
  "_routing": {
    "provider": "cloudflare",
    "model": "@cf/meta/llama-3.1-8b-instruct",
    "totalLatency": 234
  }
}
```

**Change:** Response now includes OpenAI-compatible format with routing metadata.

**Migration:** Update response parsing:

```typescript
// v0.x
const content = response.message.content;

// v1.0
const content = response.choices[0].message.content;
```

### 4. Error Handling

#### v0.x

```typescript
try {
  const result = await client.chat.create(request);
} catch (error) {
  console.error(error.message);
}
```

#### v1.0

```typescript
try {
  const result = await client.chat.completions.create(request);
} catch (error) {
  if (error instanceof ClaudeFlareError) {
    console.error(error.code, error.message);
    console.error('Request ID:', error.requestId);
  }
}
```

**Change:** Typed errors with codes and request IDs.

**Migration:** Update error handling:

```typescript
import { ClaudeFlareError, RateLimitError } from '@claudeflare/sdk';

try {
  const result = await client.chat.completions.create(request);
} catch (error) {
  if (error instanceof RateLimitError) {
    console.log('Retry after:', error.retryAfter);
  } else if (error instanceof ClaudeFlareError) {
    console.error(error.code, error.message);
  }
}
```

### 5. Provider Configuration

#### v0.x

```typescript
const client = new ClaudeFlare({
  apiKey: 'your-key',
  provider: 'cloudflare'
});
```

#### v1.0

```typescript
const client = new ClaudeFlare({
  apiKey: 'your-key',
  providers: {
    primary: 'cloudflare',
    fallback: ['groq', 'cerebras'],
    strategy: 'free-tier-first'
  }
});
```

**Change:** Multi-provider configuration with fallback.

### 6. Streaming Responses

#### v0.x

```typescript
const stream = await client.chat.stream(request);

for await (const chunk of stream) {
  console.log(chunk.text);
}
```

#### v1.0

```typescript
const stream = await client.chat.completions.stream({
  ...request,
  stream: true
});

for await (const chunk of stream) {
  if (chunk.choices[0].delta.content) {
    console.log(chunk.choices[0].delta.content);
  }
}
```

**Change:** Streaming now uses OpenAI-compatible format.

## Migration Steps

### Step 1: Update Dependencies

```bash
# Update SDKs
npm install @claudeflare/sdk@latest

# Or for Python
pip install --upgrade claudeflare

# Or for Go
go get -u github.com/claudeflare/sdk-go
```

### Step 2: Update Configuration

```typescript
// Before (v0.x)
const client = new ClaudeFlare({
  apiKey: process.env.CLOUDFLARE_API_KEY
});

// After (v1.0)
const client = new ClaudeFlare({
  apiKey: process.env.CLOUDFLARE_API_KEY,
  endpoint: 'https://api.claudeflare.com/v1',
  providers: {
    primary: 'auto',
    fallback: ['groq', 'cerebras']
  }
});
```

### Step 3: Update API Calls

```typescript
// Before (v0.x)
const result = await client.chat.create({
  messages: [{ role: 'user', content: 'Hello' }]
});

console.log(result.message.content);

// After (v1.0)
const result = await client.chat.completions.create({
  messages: [{ role: 'user', content: 'Hello' }]
});

console.log(result.choices[0].message.content);
```

### Step 4: Update Error Handling

```typescript
// Before (v0.x)
try {
  const result = await client.chat.create(request);
} catch (error) {
  console.error(error);
}

// After (v1.0)
try {
  const result = await client.chat.completions.create(request);
} catch (error) {
  if (error instanceof ClaudeFlareError) {
    switch (error.code) {
      case 'RATE_LIMITED':
        await handleRateLimit(error.retryAfter);
        break;
      case 'PROVIDER_UNAVAILABLE':
        await handleProviderUnavailable();
        break;
      default:
        console.error(error.code, error.message);
    }
  }
}
```

### Step 5: Update Streaming

```typescript
// Before (v0.x)
const stream = await client.chat.stream(request);

for await (const chunk of stream) {
  process.stdout.write(chunk.text);
}

// After (v1.0)
const stream = await client.chat.completions.stream({
  ...request,
  stream: true
});

for await (const chunk of stream) {
  if (chunk.choices[0].delta.content) {
    process.stdout.write(chunk.choices[0].delta.content);
  }
}
```

### Step 6: Test Changes

```bash
# Run tests
npm test

# Run integration tests
npm run test:integration

# Test in development environment
npm run dev
```

### Step 7: Deploy

```bash
# Deploy to staging first
npm run deploy:staging

# Test staging
npm run test:smoke -- --env staging

# Deploy to production
npm run deploy:production

# Verify production
npm run test:smoke -- --env production
```

## Compatibility Mode

If you need time to migrate, enable compatibility mode:

```typescript
const client = new ClaudeFlare({
  apiKey: 'your-key',
  compatibilityMode: 'v0',  // Use v0.x response format
  endpoint: 'https://api.claudeflare.com/v1'
});
```

**Note:** Compatibility mode will be removed in v2.0. Plan to complete migration by then.

## Rollback Plan

If issues arise:

```bash
# Rollback SDK
npm install @claudeflare/sdk@0.9.0

# Or revert code changes

# Rollback deployment
wrangler rollback --env production
```
