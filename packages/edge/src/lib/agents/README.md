# Agent Coordination System

A stateful agent orchestration system built on Cloudflare Durable Objects for distributed AI coding assistance.

## Architecture

```
User Request → Director Agent (DO)
                 ↓
           Dispatch to Planner Agents (DO)
                 ↓
           Execute via Executor Agents (DO)
                 ↓
           Aggregate & Return Response
```

## Components

### 1. Director Agent DO
Main orchestration point that:
- Receives user requests
- Dispatches to multiple Planner Agents (fan-out pattern)
- Aggregates results
- Coordinates Executor Agents
- Returns final response

**Location:** `/packages/edge/src/do/director.ts`

### 2. Planner Agent DO
Specialized agents for different expertise areas:
- **Code:** Code generation, refactoring, optimization
- **Documentation:** Documentation generation, explanation
- **Debugging:** Bug identification, fix suggestions
- **Architecture:** System design, pattern recommendations

**Location:** `/packages/edge/src/do/planner.ts`

### 3. Executor Agent DO
Executes plans generated by Planner Agents:
- Executes individual steps
- Handles errors with retry logic
- Tracks progress
- Returns final result

**Location:** `/packages/edge/src/do/executor.ts`

### 4. Agent Messenger
Inter-agent communication system:
- Point-to-point messaging
- Broadcast messaging
- Message queuing
- Delivery tracking
- Retry logic

**Location:** `/packages/edge/src/lib/agents/messenger.ts`

### 5. Agent Registry
Load balancing and health monitoring:
- Agent registration and discovery
- Health monitoring with heartbeats
- Load-based agent selection
- Automatic failover
- Metrics tracking

**Location:** `/packages/edge/src/lib/agents/registry.ts`

## Coordination Patterns

### Fan-Out Pattern
Director dispatches to multiple Planner Agents in parallel:
```typescript
const plans = await Promise.all([
  plannerCode.plan(request),
  plannerDocs.plan(request),
  plannerDebug.plan(request),
  plannerArch.plan(request),
]);
```

### Aggregation Pattern
Collect and aggregate results from multiple agents:
```typescript
const bestPlan = plans.sort((a, b) => b.priority - a.priority)[0];
```

### Fallback Pattern
If primary agent fails, use backup:
```typescript
if (primaryAgent.status === 'error') {
  return await backupAgent.execute(plan);
}
```

### Load Balancing
Distribute work across agent instances:
```typescript
const agentId = await registry.selectAgent('planner', 'code');
```

- **<50ms** agent coordination overhead

- **Sub-second** plan generation
- **Automatic** failover and recovery

## State Management

### Durable Objects Storage
Each DO persists its state:
- **Director:** Session context, active planners, completed plans, metrics
- **Planner:** Plans generated, complexity analysis, load tracking
- **Executor:** Active plan, current step, results, error count
- **Registry:** Agent info, load history, health status
- **Messenger:** Outbox, inbox, sent messages

### LRU Eviction
Director DO implements LRU eviction when approaching 128MB limit:
```typescript
while (memoryUsage + requiredBytes > 128MB) {
  evictLeastRecentlyUsed();
}
```

## Error Handling

### Retry Logic
Exponential backoff with max 3 retries:
```typescript
const backoffMs = 1000 * Math.pow(2, attempt);
```

### Recovery Strategies
- **Retry:** For transient failures
- **Fallback:** To backup agent
- **Skip:** For non-critical steps
- **Abort:** For critical failures

## Example Usage

```typescript
import { orchestrateChat } from './lib/agents';

const response = await orchestrateChat(env, {
  sessionId: 'session-123',
  userId: 'user-456',
  messages: [
    {
      role: 'user',
      content: 'Write a TypeScript function to sort an array'
    }
  ],
  context: {
    language: 'typescript',
    framework: 'react'
  }
});

console.log(response.content);
console.log(`Latency: ${response.latency}ms`);
console.log(`Agents: ${response.metadata.agentsInvolved.join(', ')}`);
```
